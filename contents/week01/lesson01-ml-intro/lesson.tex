\chapter{机器学习基本概念}
\label{chap:w01l01-ml-intro}

\begin{keypointbox}
机器学习这门课，绕来绕去其实都在做同一件事：从资料里摸出一个够用的函数。
这节先拿 YouTube 点阅预测当例子，把最基本的训练流程顺一遍：模型怎么写、loss 怎么定、参数又是怎么慢慢调出来的。
\end{keypointbox}

\section{什么是机器学习}
\label{sec:w01l01-what-is-ml}

\begin{definition}{机器学习}{w01l01-machine-learning}
机器学习研究的是如何利用资料自动找出一个函数，使它能够把输入映射到合理的输出。
\end{definition}

如果只用一句很口语的话来讲，这门课就在训练机器“自己把函数找出来”。
有些任务太复杂，没办法靠人手把规则一条条写死，像语音辨识、影像辨识、围棋落子都属于这种情况。
这时比较实际的做法不是继续硬写规则，而是把资料交给机器，让它从输入和输出之间的对应关系里，自己长出一个像样的映射。

\section{机器学习的任务类别}
\label{sec:w01l01-task-types}

课程里先按输出长什么样，把任务粗分成三类：
\begin{enumerate}
  \item \textbf{Regression（回归）}：输出是一个数值。例如根据今天和过去几天的空气品质指标，预测明天中午的 PM2.5。
  \item \textbf{Classification（分类）}：输出是在有限类别里做选择。例如垃圾邮件过滤，或从棋盘上 $19 \times 19$ 个位置中选择下一手。
  \item \textbf{Structured Learning（结构化学习）}：输出具有结构，例如句子、文档、图像等，更接近让机器“创造”一个复杂物件。
\end{enumerate}

\begin{examplebox}
回归比较像在估一个数，分类比较像在若干选项里做判断，结构化学习则更进一步，要求模型直接吐出一整个有组织的结果。
\end{examplebox}

\section{找函数的三个步骤}
\label{sec:w01l01-three-steps}

拿 YouTube 频道隔天总观看次数预测来说，真正要做的不是一上来就“猜答案”，而是把问题拆成三件比较能操作的事。

\subsection{Step 1：写出带未知参数的模型}
\label{sec:w01l01-model}

先别急着谈训练。手上如果连一个可以调的函数都没有，后面根本没东西可学。
所以第一步通常是先凭直觉写出一个最朴素的模型。以这个例子来说，很自然会猜：明天的观看人数，大概跟昨天的观看人数脱不了关系。
顺着这个想法，最简单的写法就是
\begin{equation}
  y = b + w x_1.
  \label{eq:w01l01-linear-model}
\end{equation}

\cref{eq:w01l01-linear-model} 之所以叫线性模型，是因为输出 $y$ 还是靠“特征乘上权重再加总”这种线性的方式拼出来的。
这里 $x_1$ 是输入特征（feature），也就是昨天的观看人数；$y$ 是我们想预测的目标，也就是明天的观看人数。
$w$ 叫权重（weight），决定模型对输入变化有多敏感；$b$ 叫偏差（bias），比较像整条直线的起始基准。

如果固定 $b$ 不动，$x_1$ 每多 1，预测值就跟着多 $w$；如果固定 $w$ 不动，改 $b$ 的效果则像把整条线一起往上抬或往下压。
所以这类带着未知参数、等着训练去调的函数，就是这里说的模型（model）。

\begin{examplebox}
如果某频道昨天的观看人数为 $x_1=100$，并且我们暂时取 $w=1.1$、$b=20$，那么模型会给出预测
\[
  y = 20 + 1.1 \times 100 = 130.
\]
这个例子很好用来分清两件事：只改 $b$，整条线会一起平移；只改 $w$，模型对输入的反应幅度会变。
所以 weight 管的是“斜率感”，bias 管的是“整体基准”。
\end{examplebox}

\subsection{Step 2：定义损失函数}
\label{sec:w01l01-loss}

模型写出来以后，还缺一个关键问题：到底什么叫“这组参数比较好”？
这就需要先订一个评分标准。课程里用的做法很直接，把训练资料丢进模型，得到预测值 $y$，再拿它去和真实标签 $\hat{y}$ 对照。

单笔资料的误差常见写法包括
\begin{align}
  e_{\mathrm{MAE}} &= \abs{y - \hat{y}}, \label{eq:w01l01-mae} \\
  e_{\mathrm{MSE}} &= \paren{y - \hat{y}}^2. \label{eq:w01l01-mse}
\end{align}

\subsubsection{从单点误差到整体 Loss}

\cref{eq:w01l01-mae,eq:w01l01-mse} 先讲的是单笔资料错多少。
但训练的时候，我们关心的不是某一笔有没有猜准，而是这组参数放到整份资料上到底表现得怎么样。
所以单点误差还不够，后面还得把所有样本的误差汇总起来。

把所有训练样本的误差加总并平均，就得到整体损失：
\begin{equation}
  L = \frac{1}{N}\sum_{n=1}^{N} e_n.
  \label{eq:w01l01-loss}
\end{equation}

\cref{eq:w01l01-loss} 里的 $N$ 是训练样本数，$e_n$ 是第 $n$ 笔资料的误差。
写成平均值有个实际好处：不同大小的数据集比较起来比较公平，而且 loss 不会因为样本数量一多就机械地变大。

\begin{examplebox}
假设有三笔训练资料，它们的预测误差分别是 $10$、$-5$、$20$。那么：
\begin{align*}
  \text{MAE} &= \frac{\abs{10} + \abs{-5} + \abs{20}}{3}
  = \frac{35}{3}, \\
  \text{MSE} &= \frac{10^2 + (-5)^2 + 20^2}{3}
  = \frac{525}{3} = 175.
\end{align*}
拿这三笔数据一算就很明显：MSE 会把大误差放得更大，MAE 则比较平均。
所以如果你特别在意少数样本错得很离谱，MSE 往往会更有感觉。
\end{examplebox}

\subsubsection{Loss 的几何意义}

\begin{intuitionbox}
损失函数的值越小，代表这组参数在训练资料上的表现越好。若模型只有一个参数，就可以把 $L(w)$ 画成一条曲线；若模型有两个参数，
就可以把 $L(w,b)$ 画成一张曲面或等高线图。课程里一直提到的 error surface，说白了就是“参数走到哪里，loss 就有多高”的地图。
\end{intuitionbox}

这样想训练会顺很多：我们不是在空中乱猜参数，而是在一张高低起伏的地形图上往低处走。
能把 loss 压下去，就表示参数大致走对了。

\subsection{Step 3：做优化}
\label{sec:w01l01-optimization}

到这里，问题就变成了一个优化问题：既然好坏已经能被 loss 量化，那接下来只要去找让 loss 最小的那组参数。
\begin{equation}
  w^\star, b^\star = \argmin_{w,b} L.
  \label{eq:w01l01-opt-target}
\end{equation}

这里的 $\argmin$ 要读成“找出让 loss 最小的参数位置”，不是单纯回传一个最小值数字。
也就是说，\cref{eq:w01l01-opt-target} 真正要找的是 $w^\star,b^\star$ 这一组参数。

课程里最常用的做法是梯度下降（gradient descent）。
先只看一个参数 $w$ 的情形，更新规则写成
\begin{equation}
  w^{t+1} = w^t - \eta \frac{\partial L}{\partial w}\bigg\vert_{w=w^t},
  \label{eq:w01l01-gd-update}
\end{equation}
这里的 $\eta$ 是学习率（learning rate），也就是你每次愿意跨出去多大一步；它不是从资料里学出来的，而是事先设好的超参数（hyperparameter）。

\subsubsection{为什么要减去梯度}

\cref{eq:w01l01-gd-update} 里最值得盯住的是那个负号。
直观上看，导数在回答“如果我继续把 $w$ 往右挪，loss 接下来是升还是降”：
\begin{itemize}
  \item 若 $\frac{\partial L}{\partial w} > 0$，继续往右走只会让 loss 变高，那就该往左回一点。
  \item 若 $\frac{\partial L}{\partial w} < 0$，往右走反而会下降，那就表示可以把 $w$ 往右推。
\end{itemize}

两种情况合起来，就是“沿负梯度方向走”。方向先由梯度告诉你，步子多大再交给学习率决定。

\begin{algorithm}[htbp]
  \caption{单参数梯度下降}
  \label{alg:w01l01-gradient-descent}
  \KwInput{初始参数 $w^0$，学习率 $\eta$，最大更新次数 $T$}
  \KwOutput{更新后的参数 $w^T$}
  \For{$t \leftarrow 0$ \KwTo $T-1$}{
    计算梯度 $g^t \leftarrow \dfrac{\partial L}{\partial w}\big\vert_{w=w^t}$\;
    更新参数 $w^{t+1} \leftarrow w^t - \eta g^t$\;
  }
  \KwRet{$w^T$}
\end{algorithm}

参数不只一个时，做法也没变，只是要对每个参数各看各的梯度：
\begin{align}
  w^{t+1} &= w^t - \eta \frac{\partial L}{\partial w}\bigg\vert_{w=w^t,b=b^t}, \label{eq:w01l01-gd-w} \\
  b^{t+1} &= b^t - \eta \frac{\partial L}{\partial b}\bigg\vert_{w=w^t,b=b^t}. \label{eq:w01l01-gd-b}
\end{align}

\subsubsection{学习率与更新行为}

如果把梯度理解成方向盘，那学习率就是油门。
它决定的不是“往哪走”，而是“这一步迈多大”。
\begin{itemize}
  \item 若 $\eta$ 太大，参数可能一次跳过最低点，在曲面两侧来回震荡，甚至直接发散。
  \item 若 $\eta$ 太小，loss 虽然会下降，但下降速度很慢，训练会非常耗时。
\end{itemize}

训练里经常要调学习率，原因就在这里：方向大致对了，步子不合适一样会走得很难看。

\subsubsection{单参数与多参数的关系}

\cref{eq:w01l01-gd-update} 是一维版本，\cref{eq:w01l01-gd-w,eq:w01l01-gd-b} 是二维版本。
想法完全一样，只是参数一多以后，我们就不是在一条线上左右挪，而是在更高维的参数空间里同步调整多个坐标。
把所有参数合并成向量 $\theta$ 以后，这个更新规则就会变成熟悉的统一形式
\[
  \theta^{t+1} = \theta^t - \eta \grad L\paren{\theta^t}.
\]

\begin{warningbox}
当梯度变成零时，训练会停下来。这不一定代表已经找到全域最佳解；模型也可能停在局部极小值，甚至是后续课程要讨论的鞍点附近。
\end{warningbox}

\section{线性模型的改进}
\label{sec:w01l01-linear-model}

只看前一天，通常还是太粗了。既然点阅量可能带有周期性，那把前几天一起放进来会更合理一些。
例如把一周都纳入考虑，可以写成
\begin{equation}
  y = b + \sum_{j=1}^{7} w_j x_j.
  \label{eq:w01l01-7day-model}
\end{equation}

式子变长了，不代表模型类型就变了。\cref{eq:w01l01-7day-model} 还是线性模型，因为输出照样只是“每个特征乘权重，再全部加起来”。
中间没有额外的非线性变换，也没有特征之间互相相乘，所以它只是从“看一天”变成“看七天”，骨架没变。

更一般地说，只要模型还是靠特征加权求和、最后再加偏差来出答案，它就仍属于线性模型（linear model）。
特征加多，训练误差往往会好看一点，但测试误差未必同步改善；这个落差，后面就会接到泛化和过拟合的问题上。

\begin{summarybox}
这节真正留下来的骨架其实很简单：
\begin{enumerate}
  \item 先写出一个能调参数的模型；
  \item 再决定怎样衡量它到底好不好；
  \item 最后用优化方法把参数往更好的方向推。
\end{enumerate}
后面模型会越来越复杂，但训练主线基本都跑不出这三步。
\end{summarybox}
